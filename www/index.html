Dependency Analysis Report

  1. Tangledness statistics for classes. A "tangle" is a strongly connected component of the dependency graph containing two or more nodes. The "tangledness" is given by the percentage of classes that are part of a tangle.
  2. Problematic dependencies betwen classes (CSV). This lists suggests dependencies to remove in order to remove "tangles" from the codebase. It provides one of many possible sets of dependencies to remove that would make the dependency graph acyclic, in a way to preserves the intended semantics of the dependency graph. The heuristic used to determine which dependencies should be removed is described in the paper Breaking Cycles in Noisy Hierarchies by Jiankai Sun et.al.
  3. Classes ordered by risk. The riskiness of each class is determiend using the bi-directional Page Rank score, which is the product of the normal Page Rank score, and the Page Rank score calculated for the inverse (transposed) graph. The idea is that the page rank score indicates how much a class is influenced by other classes, and the reverse page rank indicates how much it influences other classes - and a class that is both very influential, but also subject to a lot of influence, is very risky.
  4. All dependencies betwen classes (CSV). Provided for reference, and for debugging the analyzer.
  5. Tangledness statistics for namespaces. A "tangle" is a strongly connected component of the dependency graph containing two or more nodes. The "tangledness" is given by the percentage of namespaces that are part of a tangle.
  6. Problematic dependencies betwen namespaces (CSV). This lists suggests dependencies to remove in order to remove "tangles" from the codebase.
  7. Namespaces ordered by risk. The riskiness of each namespace is determiend using the bi-directional Page Rank score.
  8. All dependencies betwen namespaces (CSV). Provided for reference, and for debugging the analyzer.
